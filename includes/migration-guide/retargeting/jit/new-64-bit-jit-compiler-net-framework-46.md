### <a name="new-64-bit-jit-compiler-in-the-net-framework-46"></a><span data-ttu-id="3d272-101">Neuer 64-Bit-JIT-Compiler in .NET Framework 4.6</span><span class="sxs-lookup"><span data-stu-id="3d272-101">New 64-bit JIT compiler in the .NET Framework 4.6</span></span>

#### <a name="details"></a><span data-ttu-id="3d272-102">Details</span><span class="sxs-lookup"><span data-stu-id="3d272-102">Details</span></span>

<span data-ttu-id="3d272-103">Ab .NET Framework 4.6 wird ein neuer 64-Bit-JIT-Compiler für die Just-in-Time-Kompilierung verwendet.</span><span class="sxs-lookup"><span data-stu-id="3d272-103">Starting with the .NET Framework 4.6, a new 64-bit JIT compiler is used for just-in-time compilation.</span></span> <span data-ttu-id="3d272-104">In einigen Fällen wird eine unerwartete Ausnahme ausgelöst oder ein anderes Verhalten beobachtet als bei der Ausführung des 32-Bit-Compilers oder des älteren 64-Bit-JIT-Compilers.</span><span class="sxs-lookup"><span data-stu-id="3d272-104">In some cases, an unexpected exception is thrown or a different behavior is observed than if an app is run using the 32-bit compiler or the older 64-bit JIT compiler.</span></span> <span data-ttu-id="3d272-105">Diese Änderung wirkt sich nicht auf den 32-Bit-JIT-Compiler aus.</span><span class="sxs-lookup"><span data-stu-id="3d272-105">This change does not affect the 32-bit JIT compiler.</span></span> <span data-ttu-id="3d272-106">Die bekannten Unterschiede umfassen folgende Punkte:</span><span class="sxs-lookup"><span data-stu-id="3d272-106">The known differences include the following:</span></span>

- <span data-ttu-id="3d272-107">Unter bestimmten Umständen kann ein Unboxingvorgang in Releasebuilds mit aktivierter Optimierung eine <xref:System.NullReferenceException>-Ausnahme auslösen.</span><span class="sxs-lookup"><span data-stu-id="3d272-107">Under certain conditions, an unboxing operation may throw a <xref:System.NullReferenceException> in Release builds with optimization turned on.</span></span>
- <span data-ttu-id="3d272-108">In manchen Fällen kann bei der Ausführung von Produktionscode in einem großen Methodentext eine <xref:System.StackOverflowException>-Ausnahme ausgelöst werden.</span><span class="sxs-lookup"><span data-stu-id="3d272-108">In some cases, execution of production code in a large method body may throw a <xref:System.StackOverflowException>.</span></span>
- <span data-ttu-id="3d272-109">Unter bestimmten Bedingungen werden in Releasebuilds an eine Methode übergebene Strukturen als Verweistypen statt als Werttypen behandelt.</span><span class="sxs-lookup"><span data-stu-id="3d272-109">Under certain conditions, structures passed to a method are treated as reference types rather than as value types in Release builds.</span></span> <span data-ttu-id="3d272-110">Eins der Anzeichen dieses Problems besteht darin, dass die einzelnen Elemente einer Sammlung in unerwarteter Reihenfolge angezeigt werden.</span><span class="sxs-lookup"><span data-stu-id="3d272-110">One of the manifestations of this issue is that the individual items in a collection appear in an unexpected order.</span></span>
- <span data-ttu-id="3d272-111">Unter bestimmten Bedingungen ist der Vergleich von <xref:System.UInt16>-Werten mit festgelegtem hohem Bit fehlerhaft, wenn Optimierung aktiviert ist.</span><span class="sxs-lookup"><span data-stu-id="3d272-111">Under certain conditions, the comparison of <xref:System.UInt16> values with their high bit set is incorrect if optimization is enabled.</span></span>
- <span data-ttu-id="3d272-112">Unter bestimmten Umständen, insbesondere beim Initialisieren von Arraywerten, kann die Speicherinitialisierung durch die IL-Anweisung <xref:System.Reflection.Emit.OpCodes.Initblk?displayProperty=nameWithType> mit einem falschen Wert erfolgen.</span><span class="sxs-lookup"><span data-stu-id="3d272-112">Under certain conditions, particularly when initializing array values, memory initialization by the <xref:System.Reflection.Emit.OpCodes.Initblk?displayProperty=nameWithType> IL instruction may initialize memory with an incorrect value.</span></span> <span data-ttu-id="3d272-113">Dies kann entweder zu einem Ausnahmefehler oder zu einer falschen Ausgabe führen.</span><span class="sxs-lookup"><span data-stu-id="3d272-113">This can result either in an unhandled exception or incorrect output.</span></span>
- <span data-ttu-id="3d272-114">Unter bestimmten seltenen Bedingungen kann ein bedingter Bittest den falschen <xref:System.Boolean>-Wert zurückgeben oder eine Ausnahme auslösen, wenn Compileroptimierungen aktiviert sind.</span><span class="sxs-lookup"><span data-stu-id="3d272-114">Under certain rare conditions, a conditional bit test can return the incorrect <xref:System.Boolean> value or throw an exception if compiler optimizations are enabled.</span></span>
- <span data-ttu-id="3d272-115">Wenn unter bestimmten Umständen eine `if`-Anweisung für die Prüfung auf eine Bedingung vor dem Eintritt in einen `try`-Block oder beim Verlassen eines `try`-Blocks erfolgt und die gleiche Bedingung im `catch`- oder `finally`-Block ausgewertet wird, entfernt der neue 64-Bit-JIT-Compiler beim Optimieren von Code die `if`-Bedingung aus dem `catch`- oder `finally`-Block.</span><span class="sxs-lookup"><span data-stu-id="3d272-115">Under certain conditions, if an `if` statement is used to test for a condition before entering  a `try` block and in the exit from the `try` block, and the same condition is evaluated in the `catch` or `finally` block, the new 64-bit JIT compiler removes the `if` condition from the `catch` or `finally` block when it optimizes code.</span></span> <span data-ttu-id="3d272-116">Daher wird Code innerhalb der `if`-Anweisung im `catch`- oder `finally`-Block ohne Bedingung ausgeführt.</span><span class="sxs-lookup"><span data-stu-id="3d272-116">As a result, code inside the `if` statement in the `catch` or `finally` block is executed unconditionally.</span></span>
