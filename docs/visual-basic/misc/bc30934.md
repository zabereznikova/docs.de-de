# <a name="conversion-from-39lttype1gt39-to-39lttype2gt39-cannot-occur-in-a-constant-expression-used-as-an-argument-to-an-attribute"></a><span data-ttu-id="a9208-102">Konvertierung von &#39; &lt;Typ1&gt;&#39; in &#39;&lt; Typ2&gt;&#39; kann nicht in einem konstanten Ausdruck als Argument für ein Attribut verwendet auftreten</span><span class="sxs-lookup"><span data-stu-id="a9208-102">Conversion from &#39;&lt;type1&gt;&#39; to &#39;&lt;type2&gt;&#39; cannot occur in a constant expression used as an argument to an attribute</span></span>
<span data-ttu-id="a9208-103">Ein für ein Attributargument verwendeter Ausdruck wird als Datentyp ausgewertet, der sich von dem des entsprechenden Attributparameters unterscheidet, und [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] lässt die erforderliche Typkonvertierung für Attributargumente nicht zu.</span><span class="sxs-lookup"><span data-stu-id="a9208-103">An expression used for an attribute argument evaluates to a data type different from that of the corresponding attribute parameter, and [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] does not allow the required type conversion for attribute arguments.</span></span>  
  
 <span data-ttu-id="a9208-104">Ein Attribut stellt Metadaten für das Element bereit, auf das es angewendet wird, und der Compiler muss zur Kompilierzeit in der Lage sein, alle Metadaten zu erstellen.</span><span class="sxs-lookup"><span data-stu-id="a9208-104">An attribute provides metadata for the element it is applied to, and the compiler must be able to construct all the metadata at compile time.</span></span> <span data-ttu-id="a9208-105">Daher muss jedes Attribut Werte verwenden, die zur Kompilierzeit konstant sind, und jedes Attributargument muss als ein zur Kompilierzeit konstanter Wert ausgewertet werden.</span><span class="sxs-lookup"><span data-stu-id="a9208-105">For this reason, every attribute must use values that are constant at compile time, and therefore every attribute argument must evaluate to a compile-time constant value.</span></span>  
  
 <span data-ttu-id="a9208-106">Bei bestimmten Typkonvertierungen können keine Werte erzeugt werden, die zur Kompilierzeit konstant sind.</span><span class="sxs-lookup"><span data-stu-id="a9208-106">Certain type conversions cannot produce values that are constant at compile time.</span></span> <span data-ttu-id="a9208-107">Ob beispielsweise `String` in `Double` oder `Date` konvertiert wird, hängt von der Gebietsschemaeinstellung zur Laufzeit ab.</span><span class="sxs-lookup"><span data-stu-id="a9208-107">For example, converting a `String` to a `Double` or a `Date` depends on the locale setting at run time.</span></span> <span data-ttu-id="a9208-108">Andere Konvertierungen (z. B. die Konvertierung eines Array eines abgeleiteten Typs in ein `Object`-Array) verursachen eine Vielzahl von Problemen, durch die der Compiler sie für Attributargumente nicht zulassen kann.</span><span class="sxs-lookup"><span data-stu-id="a9208-108">Other conversions, such as an array of a derived type to an array of `Object`, present a variety of problems that do not permit the compiler to allow them on attribute arguments.</span></span>
