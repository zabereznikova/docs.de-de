## <a name="general-guidelines"></a><span data-ttu-id="bee61-169">Allgemeine Richtlinien</span><span class="sxs-lookup"><span data-stu-id="bee61-169">General Guidelines</span></span>

<span data-ttu-id="bee61-170">Wenngleich es weiterhin vorzuziehen ist, Funktionalität durch Änderung des Codes eines Objekts oder durch Ableitung eines neuen Typs hinzuzufügen – sofern dies sinnvoll und möglich ist –, sind Erweiterungsmethoden zu einer wichtigen Option zum Bereitstellen wiederverwendbarer Funktionalität im gesamten .NET-Ökosystem geworden.</span><span class="sxs-lookup"><span data-stu-id="bee61-170">While it's still considered preferable to add functionality by modifying an object's code or deriving a new type whenever it's reasonable and possible to do so, extension methods have become a crucial option for creating reusable functionality throughout the .NET ecosystem.</span></span> <span data-ttu-id="bee61-171">In Fällen, in denen die ursprüngliche Quelle nicht Ihrer Kontrolle unterliegt, ein abgeleitetes Objekt ungeeignet ist oder nicht verwendet werden kann oder die Funktionalität nicht über den anwendbaren Bereich hinaus offengelegt werden soll, stellen Erweiterungsmethoden eine ausgezeichnete Wahl dar.</span><span class="sxs-lookup"><span data-stu-id="bee61-171">For those occasions when the original source isn't under your control, when a derived object is inappropriate or impossible, or when the functionality shouldn't be exposed beyond its applicable scope, Extension methods are an excellent choice.</span></span>

<span data-ttu-id="bee61-172">Weitere Informationen zu abgeleiteten Typen finden Sie unter [Vererbung](./inheritance.md).</span><span class="sxs-lookup"><span data-stu-id="bee61-172">For more information on derived types, see [Inheritance](./inheritance.md).</span></span>

<span data-ttu-id="bee61-173">Wenn Sie eine Erweiterungsmethode zum Erweitern eines Typs verwenden, dessen Quellcode Sie nicht ändern können, laufen Sie Gefahr, dass eine Änderung an der Implementierung des Typs zu einer Beschädigung Ihrer Erweiterungsmethode führt.</span><span class="sxs-lookup"><span data-stu-id="bee61-173">When using an extension method to extend a type whose source code you aren't in control of, you run the risk that a change in the implementation of the type will cause your extension method to break.</span></span>

<span data-ttu-id="bee61-174">Wenn Sie Erweiterungsmethoden für einen gegebenen Typ implementieren, beachten Sie die folgenden Punkte:</span><span class="sxs-lookup"><span data-stu-id="bee61-174">If you do implement extension methods for a given type, remember the following points:</span></span>

- <span data-ttu-id="bee61-176">Erweiterungsmethoden werden auf Namespace-Ebene eingebunden.</span><span class="sxs-lookup"><span data-stu-id="bee61-176">Extension methods are brought into scope at the namespace level.</span></span> <span data-ttu-id="bee61-177">Wenn Sie z.
