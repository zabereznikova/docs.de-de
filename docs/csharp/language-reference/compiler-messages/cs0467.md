# <a name="compiler-warning-level-2-cs0467"></a><span data-ttu-id="c5ea8-102">Compilerwarnung (Stufe 2) CS0467</span><span class="sxs-lookup"><span data-stu-id="c5ea8-102">Compiler Warning (level 2) CS0467</span></span>
<span data-ttu-id="c5ea8-103">Mehrdeutigkeit zwischen der Methode „method“ und der Nichtmethode „non-method“.</span><span class="sxs-lookup"><span data-stu-id="c5ea8-103">Ambiguity between method 'method' and non-method 'non-method'.</span></span> <span data-ttu-id="c5ea8-104">Verwenden Sie die Methodengruppe.</span><span class="sxs-lookup"><span data-stu-id="c5ea8-104">Using method group.</span></span>

## <a name="example"></a><span data-ttu-id="c5ea8-106">Beispiel</span><span class="sxs-lookup"><span data-stu-id="c5ea8-106">Example</span></span>  
 <span data-ttu-id="c5ea8-107">Im folgenden Beispiel wird der Fehler CS0467 generiert.</span><span class="sxs-lookup"><span data-stu-id="c5ea8-107">The following example generates CS0467.</span></span>  
  
```csharp  
// CS0467.cs  
interface IList
{  
    int Count { get; set; }  
}  
  
interface ICounter  
{  
    void Count(int i);  
}  
  
interface IListCounter : IList, ICounter {}  
  
class Driver
{  
    void Test(IListCounter x)  
    {  
        // The following line causes the warning. The assignment also  
        // causes an error because you can't assign a value to a method.  
        x.Count = 1;  
        x.Count(3);
        // To resolve the warning, you can change the name of the method or
        // the property.  
  
        // You also can disambiguate by specifying IList or ICounter.  
        ((IList)x).Count = 1;  
        ((ICounter)x).Count(3);  
    }  
  
    static void Main()
    {  
    }  
}  
```
