# Compiler Error CS1009
Nicht erkannte Escapesequenz.  
  
 Ein unerwartetes Zeichen folgt einem umgekehrten Schr채gstrich \(\\\) in [Zeichenfolge](../../../csharp/language-reference/keywords/string.md).  Der Compiler erwartet eines der g체ltigen Escapezeichen.  Weitere Informationen finden Sie unter [Escapezeichen](../Topic/Character%20Escapes%20in%20Regular%20Expressions.md).  
  
 Im folgenden Beispiel wird CS1009 generiert.  
  
```c#  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";     
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 Eine h채ufige Ursache dieses Fehlers verwendet umgekehrten Schr채gstriches in einem Dateinamen, wie im folgenden Beispiel gezeigt.  
  
```c#

```  
  
 Um diesen Fehler zu beheben, verwenden Sie "\\ \\" oder das Zeichenfolgenliteral mit, wie im folgenden Beispiel gezeigt.  
  
```c#  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";     
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
