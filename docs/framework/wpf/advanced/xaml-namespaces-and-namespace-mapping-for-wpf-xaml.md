## <a name="wpf-and-assembly-loading"></a><span data-ttu-id="d781b-162">WPF und Laden von Assemblys</span><span class="sxs-lookup"><span data-stu-id="d781b-162">WPF and Assembly Loading</span></span>  
 <span data-ttu-id="d781b-163">Der XAML-Schema Kontext für WPF ist in das WPF-Anwendungsmodell integriert, das wiederum das CLR-definierte Konzept der <xref:System.AppDomain>verwendet.</span><span class="sxs-lookup"><span data-stu-id="d781b-163">The XAML schema context for WPF integrates with the WPF application model, which in turn uses the CLR-defined concept of <xref:System.AppDomain>.</span></span> <span data-ttu-id="d781b-164">In der folgenden Sequenz wird beschrieben, wie der XAML-Schema Kontext interpretiert, wie Assemblys geladen oder Typen zur Laufzeit oder zur Entwurfszeit gefunden werden, basierend auf der WPF-Verwendung von <xref:System.AppDomain> und anderen Faktoren.</span><span class="sxs-lookup"><span data-stu-id="d781b-164">The following sequence describes how XAML schema context interprets how to either load assemblies or find types at run time or design time, based on the WPF use of <xref:System.AppDomain> and other factors.</span></span>  
  
1. <span data-ttu-id="d781b-165">Durchlaufen Sie die <xref:System.AppDomain>, und suchen Sie nach einer bereits geladenen Assembly, die mit allen Aspekten des Namens übereinstimmt, beginnend mit der zuletzt geladenen Assembly.</span><span class="sxs-lookup"><span data-stu-id="d781b-165">Iterate through the <xref:System.AppDomain>, looking for an already-loaded assembly that matches all aspects of the name, starting from the most recently loaded assembly.</span></span>  
  
2. <span data-ttu-id="d781b-166">Wenn der Name qualifiziert ist, wenden Sie <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> auf den qualifizierten Namen an.</span><span class="sxs-lookup"><span data-stu-id="d781b-166">If the name is qualified, call <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> on the qualified name.</span></span>  
  
3. <span data-ttu-id="d781b-167">Wenn der Kurzname und das öffentliche Schlüsseltoken eines qualifizierten Namens mit der Assembly übereinstimmen, aus der das Markup geladen wurde, gib diese Assembly zurück.</span><span class="sxs-lookup"><span data-stu-id="d781b-167">If the short name + public key token of a qualified name matches the assembly that the markup was loaded from, return that assembly.</span></span>  
  
4. <span data-ttu-id="d781b-168">Verwenden Sie den Kurznamen und das öffentliche Schlüssel Token, um <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType>aufzurufen.</span><span class="sxs-lookup"><span data-stu-id="d781b-168">Use the short name + public key token to call <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType>.</span></span>  
  
5. <span data-ttu-id="d781b-169">Wenn der Name nicht qualifiziert ist, wenden Sie sich an <xref:System.Reflection.Assembly.LoadWithPartialName%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="d781b-169">If the name is unqualified, call <xref:System.Reflection.Assembly.LoadWithPartialName%2A?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="d781b-170">Loose XAML verwendet Schritt 3 nicht, da es nicht aus Assemblys geladen wird.</span><span class="sxs-lookup"><span data-stu-id="d781b-170">Loose XAML does not use Step 3; there is no loaded-from assembly.</span></span>  
  
 <span data-ttu-id="d781b-171">Das kompilierte XAML für WPF (generiert über xamlbuildtask) verwendet nicht die bereits geladenen Assemblys aus <xref:System.AppDomain> (Schritt 1).</span><span class="sxs-lookup"><span data-stu-id="d781b-171">Compiled XAML for WPF (generated via XamlBuildTask) does not use the already-loaded assemblies from <xref:System.AppDomain> (Step 1).</span></span> <span data-ttu-id="d781b-172">Darüber hinaus sollte der aus XamlBuildTask resultierende Name nie unqualifiziert sein, weshalb Schritt 5 hier nicht anwendbar ist.</span><span class="sxs-lookup"><span data-stu-id="d781b-172">Also, the name should never be unqualified from XamlBuildTask output, so Step 5 does not apply.</span></span>
