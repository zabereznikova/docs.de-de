## <a name="hosting-workflows-in-iis-alone"></a><span data-ttu-id="b3b1b-109">Hosten von Workflows nur in IIS</span><span class="sxs-lookup"><span data-stu-id="b3b1b-109">Hosting workflows in IIS alone</span></span>

> [!WARNING]
>  <span data-ttu-id="b3b1b-112">Von [!INCLUDE[iisver](../../../includes/iisver-md.md)] werden Anwendungspools regelmäßig aus verschiedenen Gründen wiederverwendet.</span><span class="sxs-lookup"><span data-stu-id="b3b1b-112">[!INCLUDE[iisver](../../../includes/iisver-md.md)] recycles application pools periodically for various reasons.</span></span> <span data-ttu-id="b3b1b-113">Wenn ein Anwendungspool wiederverwendet wird, akzeptiert IIS keine an den alten Pool gerichteten Nachrichten und instanziiert einen neuen Anwendungspool, um neue Anforderungen zu akzeptieren.</span><span class="sxs-lookup"><span data-stu-id="b3b1b-113">When an application pool is recycled, IIS stops accepting messages to the old pool, and instantiates a new application pool to accept new requests.</span></span> <span data-ttu-id="b3b1b-114">Wenn ein Workflow nach dem Senden einer Antwort weiter ausgeführt wird, hat [!INCLUDE[iisver](../../../includes/iisver-md.md)] davon keine Kenntnis, und der Hostanwendungspool wird möglicherweise wiederverwendet.</span><span class="sxs-lookup"><span data-stu-id="b3b1b-114">If a workflow continues working after sending a response, [!INCLUDE[iisver](../../../includes/iisver-md.md)] will not be aware of the work being done, and may recycle the hosting application pool.</span></span> <span data-ttu-id="b3b1b-115">Wenn dies geschieht, der Workflow abgebrochen wird und Überwachungsdienste aufzeichnen wird eine [1004 - WorkflowInstanceAborted](../../../docs/framework/windows-workflow-foundation/1004-workflowinstanceaborted.md) Nachricht mit einem leeren Feld "Grund".</span><span class="sxs-lookup"><span data-stu-id="b3b1b-115">If this happens, the workflow will abort, and tracking services will record a [1004 - WorkflowInstanceAborted](../../../docs/framework/windows-workflow-foundation/1004-workflowinstanceaborted.md) message with an empty Reason field.</span></span>  
>

>  <span data-ttu-id="b3b1b-117">Bei Verwendung von AppFabric setzt der Workflowverwaltungsdienst die Ausführung des Workflows schließlich ab dem letzten erfolgreichen Persistenzpunkt fort, wenn Persistenz verwendet wird.</span><span class="sxs-lookup"><span data-stu-id="b3b1b-117">If AppFabric is used, the workflow management service will eventually resume the workflow from the last successful persistence point if persistence is used.</span></span> <span data-ttu-id="b3b1b-118">Wenn keine Persistenz verwendet wird und der Workflow Vorgänge außerhalb eines Anforderungs-/Wartemusters ausführt, gehen die Daten beim Workflowabbruch verloren.</span><span class="sxs-lookup"><span data-stu-id="b3b1b-118">If no persistence is used, and the workflow performs operations outside a Request/Response pattern, data will be lost when the workflow aborts.</span></span>
