 <span data-ttu-id="f372f-110">Die Verwendung von Anschlussnummern zur Unterscheidung zwischen Anwendungen barg Sicherheitsprobleme.</span><span class="sxs-lookup"><span data-stu-id="f372f-110">Using port numbers to distinguish between applications had security problems.</span></span> <span data-ttu-id="f372f-111">Firewalls sind in der Regel so konfiguriert, dass Sie TCP-Verkehr an allen Anschlüssen außer den wenigen wohlbekannten Eingangspunkten blockieren, sodass die Bereitstellung einer Anwendung, die einen Nicht-Standardanschluss verwendet, durch das Vorhandensein firmenspezifischer oder persönlicher Firewalls oft kompliziert oder sogar unmöglich ist.</span><span class="sxs-lookup"><span data-stu-id="f372f-111">Firewalls are generally configured to block TCP traffic on all ports except for a few well-known entry points, so deploying an application that uses a non-standard port is often complicated or even impossible due to the presence of corporate and personal firewalls.</span></span> <span data-ttu-id="f372f-112">Anwendungen, die über bekannten, bereits zugelassene Standardanschlüsse kommunizieren können, verringern die Angriffsfläche für externe Angriffe.</span><span class="sxs-lookup"><span data-stu-id="f372f-112">Applications that can communicate over standard, well-known ports that are already permitted, reduce the external attack surface.</span></span> <span data-ttu-id="f372f-113">Viele Netzwerkanwendungen verwenden das HTTP-Protokoll, da die meisten Firewalls standardmäßig so konfiguriert sind, dass sie Verkehr am TCP-Anschluss&#160;80 zulassen.</span><span class="sxs-lookup"><span data-stu-id="f372f-113">Many network applications make use of the HTTP protocol because most firewalls are configured by default to allow traffic on TCP port 80.</span></span>  
  
 <span data-ttu-id="f372f-114">Das HTTP.SYS-Modell, in dem der Verkehr für viele verschiedene HTTP-Anwendungen als Multiplex über einen einzelnen TCP-Anschluss geleitet wird, ist heute Standard auf der Windows-Plattform.</span><span class="sxs-lookup"><span data-stu-id="f372f-114">The HTTP.SYS model in which traffic for many different HTTP applications is multiplexed onto a single TCP port has become standard on the Windows platform.</span></span> <span data-ttu-id="f372f-115">Dies bietet Firewalladministratoren einen gemeinsamen Steuerungspunkt, und bietet zugleich Anwendungsentwicklern die Möglichkeit, die Bereitstellungskosten beim Erstellen neuer Anwendungen, die das Netzwerk verwenden können, zu minimieren.</span><span class="sxs-lookup"><span data-stu-id="f372f-115">This provides a common point of control for firewall administrators while allowing application developers to minimize the deployment cost of building new applications that can make use of the network.</span></span>  
  
 <span data-ttu-id="f372f-116">Die Möglichkeit, Anschlüsse für mehrere HTTP-Anwendungen freizugeben ist schon lange eine Funktion von Internetinformationsdiensten (Internet Information Services, IIS).</span><span class="sxs-lookup"><span data-stu-id="f372f-116">The ability to share ports across multiple HTTP applications has long been a feature of Internet Information Services (IIS).</span></span> <span data-ttu-id="f372f-117">Allerdings war es nur mit der Einführung von http. ## <a name="port-sharing-architecture"></a><span data-ttu-id="f372f-121">Architektur der Anschlussfreigabe</span><span class="sxs-lookup"><span data-stu-id="f372f-121">Port Sharing Architecture</span></span>  
 <span data-ttu-id="f372f-122">Die Architektur der Port Freigabe in WCF besteht aus drei Hauptkomponenten:</span><span class="sxs-lookup"><span data-stu-id="f372f-122">The Port Sharing architecture in WCF has three main components:</span></span>  
  
- <span data-ttu-id="f372f-123">Ein Arbeitsprozess: Ein beliebiger Prozess, der mithilfe freigegebener Anschlüsse über net.tcp:// kommuniziert.</span><span class="sxs-lookup"><span data-stu-id="f372f-123">A Worker Process: Any process communicating over net.tcp:// using shared ports.</span></span>  
  
- <span data-ttu-id="f372f-124">Der WCF-TCP-Transport: Implementiert das net. TCP://-Protokoll.</span><span class="sxs-lookup"><span data-stu-id="f372f-124">The WCF TCP transport: Implements the net.tcp:// protocol.</span></span>  
  
- <span data-ttu-id="f372f-125">Der Net.TCP-Portfreigabedienst: Ermöglicht mehreren Arbeitsprozessen, den gleichen TCP-Anschluss zu verwenden.</span><span class="sxs-lookup"><span data-stu-id="f372f-125">The Net.TCP Port Sharing Service: Allows many worker processes to share the same TCP port.</span></span>  
  
 <span data-ttu-id="f372f-126">Der Net.TCP-Portfreigabedienst ist ein Windows-Dienst im Benutzermodus, der net.tcp://-Verbindungen im Namen von Arbeitsprozessen akzeptiert, die sich über ihn verbinden.</span><span class="sxs-lookup"><span data-stu-id="f372f-126">The Net.TCP Port Sharing Service is a user-mode Windows service that accepts net.tcp:// connections on behalf of the worker processes that connect through it.</span></span> <span data-ttu-id="f372f-127">Wenn eine Socketverbindung ankommt, untersucht der Portfreigabedienst den eingehenden Nachrichtenstrom, um dessen Zieladresse zu erhalten.</span><span class="sxs-lookup"><span data-stu-id="f372f-127">When a socket connection arrives, the port sharing service inspects the incoming message stream to obtain its destination address.</span></span> <span data-ttu-id="f372f-128">Basierend auf dieser Adresse kann der Portfreigabedienst den Datenstrom zu der Anwendung weiterleiten, die ihn letztendlich verarbeitet.</span><span class="sxs-lookup"><span data-stu-id="f372f-128">Based on this address, the port sharing service can route the data stream to the application that ultimately processes it.</span></span>  
  
 <span data-ttu-id="f372f-129">Wenn ein WCF-Dienst, der die net.
