## <a name="protocols"></a><span data-ttu-id="29e8e-104">Protokolle</span><span class="sxs-lookup"><span data-stu-id="29e8e-104">Protocols</span></span>  

 <span data-ttu-id="29e8e-105">Deaktivieren Sie die SOAP 1.2-Unterstützung von ASP.NET 2.0:</span><span class="sxs-lookup"><span data-stu-id="29e8e-105">Disable ASP.NET 2.0’s support for SOAP 1.2:</span></span>

```xml  
      <webServices >  
          <protocols>
```  

 <span data-ttu-id="29e8e-106">Dies ist empfehlenswert, da für WCF Nachrichten erforderlich sind, die mit verschiedenen Protokollen wie SOAP 1,1 und SOAP 1,2 konform sind, um unterschiedliche Endpunkte zu verwenden.</span><span class="sxs-lookup"><span data-stu-id="29e8e-106">Doing so is advisable because WCF requires messages conforming to different protocols, like SOAP 1.1 and SOAP 1.2, to go by using different endpoints.</span></span> <span data-ttu-id="29e8e-107">Wenn ein ASP.NET 2,0-Webdienst für die Unterstützung von SOAP 1,1 und SOAP 1,2 (der Standardkonfiguration) konfiguriert ist, kann er nicht zu einem einzelnen WCF-Endpunkt an der ursprünglichen Adresse migriert werden, der sicherlich mit allen vorhandenen Clients des ASP.NET-Webdiensts kompatibel wäre.</span><span class="sxs-lookup"><span data-stu-id="29e8e-107">If an ASP.NET 2.0 Web service is configured to support both SOAP 1.1 and SOAP 1.2, which is the default configuration, then it cannot be migrated forward to a single WCF endpoint at the original address that would be certainly be compatible with all of the ASP.NET Web service’s existing clients.</span></span> <span data-ttu-id="29e8e-108">Auch das Auswählen von SOAP 1.2 anstelle von 1.1 hat eine stärkere Einschränkung der Clients für den Dienst zur Folge.</span><span class="sxs-lookup"><span data-stu-id="29e8e-108">Also choosing SOAP 1.2 instead of 1.1 will more severely restrict the clientele of the service.</span></span>

## <a name="exception-handling"></a><span data-ttu-id="29e8e-130">Ausnahmebehandlung</span><span class="sxs-lookup"><span data-stu-id="29e8e-130">Exception Handling</span></span>  

 <span data-ttu-id="29e8e-131">Entwerfen Sie beim Ausarbeiten der Strukturen von Datentypen, die von einem Dienst gesendet und empfangen werden sollen, auch Strukturen für die verschiedenen Ausnahmetypen, die in einem an einen Client zu übermittelnden Dienst auftreten können.</span><span class="sxs-lookup"><span data-stu-id="29e8e-131">In designing the structures of the data types to be sent and received by a service, also design structures to represent the various types of exceptions that might occur within a service that one might wish to convey to a client.</span></span>  
  
```csharp  
[Serializable]  
[XmlRoot(Namespace="ExplicitNamespace", IsNullable=true)]  
public partial class AnticipatedException
{
    private string anticipatedExceptionInformationField;  

    public string AnticipatedExceptionInformation
    {  
        get {
            return this.anticipatedExceptionInformationField;  
        }  
        set {  
            this.anticipatedExceptionInformationField = value;  
        }  
    }  
}  
```  
  
 <span data-ttu-id="29e8e-132">Konfigurieren Sie Klassen dieser Art so, dass sie eigenständig zu XML serialisiert werden können:</span><span class="sxs-lookup"><span data-stu-id="29e8e-132">Give such classes the ability to serialize themselves to XML:</span></span>  
  
```csharp  
public XmlNode ToXML()  
{  
     XmlSerializer serializer = new XmlSerializer(  
      typeof(AnticipatedException));  
     MemoryStream memoryStream = new MemoryStream();  
     XmlTextWriter writer = new XmlTextWriter(  
     memoryStream, UnicodeEncoding.UTF8);  
     serializer.Serialize(writer, this);  
     XmlDocument document = new XmlDocument();  
     document.LoadXml(new string(  
     UnicodeEncoding.UTF8.GetChars(  
     memoryStream.GetBuffer())).Trim());  
    return document.DocumentElement;  
}  
```  
  
 <span data-ttu-id="29e8e-133">Die Klassen können anschließend zum Angeben der Details für explizit ausgelöste <xref:System.Web.Services.Protocols.SoapException>-Instanzen verwendet werden:</span><span class="sxs-lookup"><span data-stu-id="29e8e-133">The classes can then be used to provide the details for explicitly thrown <xref:System.Web.Services.Protocols.SoapException> instances:</span></span>

```csharp  
     "Fault occurred",  
     SoapException.ClientFaultCode,  
     Context.Request.Url.AbsoluteUri,  
     exception.ToXML());  
```  
  
 <span data-ttu-id="29e8e-134">Diese Ausnahme Klassen können problemlos mit der WCF-Klasse wieder verwendet werden <xref:System.ServiceModel.FaultException%601> , um eine neue `FaultException<AnticipatedException>(anticipatedException);`</span><span class="sxs-lookup"><span data-stu-id="29e8e-134">These exception classes will be readily reusable with the WCF <xref:System.ServiceModel.FaultException%601> class to throw a new `FaultException<AnticipatedException>(anticipatedException);`</span></span>
