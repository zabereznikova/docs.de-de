 <span data-ttu-id="cb24b-108">Auf der mittleren Ebene werden Entitäten vom Datenkontext erstellt, der deren Zustand nachverfolgt und das verzögerte Laden von Änderungen aus der Datenbank und das class="sxs-lookup"><span data-stu-id="cb24b-109">These entities are "attached" to the `DataContext`.</span></span> <span data-ttu-id="cb24b-110">Nachdem die Entitäten jedoch mittels Serialisierung an <span data-ttu-id="cb24b-111">Entitäten, die vom Client zum Update zurückgesendet werden, müssen erneut an den Datenkontext angefügt werden, bevor die Änderungen von Parallelität benötigt werden.</span><span class="sxs-lookup"><span data-stu-id="cb24b-112">The client is responsible for providing original values and/or timestamps back to the middle tier if those
