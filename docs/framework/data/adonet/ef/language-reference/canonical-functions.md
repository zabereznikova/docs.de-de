# <a name="canonical-functions"></a><span data-ttu-id="43dc3-102">Kanonische Funktionen</span><span class="sxs-lookup"><span data-stu-id="43dc3-102">Canonical Functions</span></span>
<span data-ttu-id="43dc3-103">In diesem Abschnitt werden die von allen Datenanbietern unterstützten und in allen Abfragetechnologien verwendbaren kanonischen Funktionen erläutert.</span><span class="sxs-lookup"><span data-stu-id="43dc3-103">This section discusses canonical functions that are supported by all data providers, and can be used by all querying technologies.</span></span> <span data-ttu-id="43dc3-104">Kanonische Funktionen können von Anbietern nicht erweitert werden.</span><span class="sxs-lookup"><span data-stu-id="43dc3-104">Canonical functions cannot be extended by a provider.</span></span>  
  
 <span data-ttu-id="43dc3-105">Diese kanonischen Funktionen werden in die entsprechenden Datenquellenfunktionen der Anbieter übersetzt.</span><span class="sxs-lookup"><span data-stu-id="43dc3-105">These canonical functions will be translated to the corresponding data source functionality for the provider.</span></span> <span data-ttu-id="43dc3-106">Dadurch können Funktionsaufrufe für Datenquellen in einer allgemeinen Form ausgedrückt werden.</span><span class="sxs-lookup"><span data-stu-id="43dc3-106">This allows for function invocations expressed in a common form across data sources.</span></span>  
  
 <span data-ttu-id="43dc3-107">Da diese kanonischen Funktionen datenquellenunabhängig sind, werden die Argument- und Rückgabetypen kanonischer Funktionen in Form von Typen im konzeptionellen Modell definiert.</span><span class="sxs-lookup"><span data-stu-id="43dc3-107">Because these canonical functions are independent of data sources, argument and return types of canonical functions are defined in terms of types in the conceptual model.</span></span> <span data-ttu-id="43dc3-108">Jedoch werden von einigen Datenquellen möglicherweise nicht alle Typen im konzeptionellen Modell unterstützt.</span><span class="sxs-lookup"><span data-stu-id="43dc3-108">However, some data sources might not support all types in the conceptual model.</span></span>  
  
 <span data-ttu-id="43dc3-109">Wenn in einer [!INCLUDE[esql](../../../../../../includes/esql-md.md)]-Abfrage kanonische Funktionen verwendet werden, werden in Datenquellen die entsprechenden Funktionen aufgerufen.</span><span class="sxs-lookup"><span data-stu-id="43dc3-109">When canonical functions are used in an [!INCLUDE[esql](../../../../../../includes/esql-md.md)] query, the appropriate function will be called at the data source.</span></span>  
  
 <span data-ttu-id="43dc3-110">Für alle kanonischen Funktionen sind das Verhalten bei NULL-Eingaben sowie die Fehlerbedingungen explizit angegeben.</span><span class="sxs-lookup"><span data-stu-id="43dc3-110">All canonical functions have both null-input behavior and error conditions explicitly specified.</span></span> <span data-ttu-id="43dc3-111">Speicheranbieter sollten diesen Vorgaben entsprechend vorgehen. Dieses Verhalten wird vom [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] jedoch nicht erzwungen.</span><span class="sxs-lookup"><span data-stu-id="43dc3-111">Store providers should comply with that behavior, but [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] does not enforce this behavior.</span></span>  
  
 <span data-ttu-id="43dc3-112">LINQ-Szenarios, Abfragen für die [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] bei CLR-Methoden den Methoden in der zugrunde liegenden Datenquelle zugeordnet werden.</span><span class="sxs-lookup"><span data-stu-id="43dc3-112">For LINQ scenarios, queries against the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] involve mapping CLR methods to methods in the underlying data source.</span></span> <span data-ttu-id="43dc3-113">Die CLR-Methoden werden kanonischen Funktionen zugeordnet, sodass bestimmte Methoden unabhängig von der Datenquelle ordnungsgemäß zugeordnet werden.</span><span class="sxs-lookup"><span data-stu-id="43dc3-113">The CLR methods map to canonical functions, so that a specific set of methods will correctly map, regardless of the data source.</span></span>  
  
## <a name="canonical-functions-namespace"></a><span data-ttu-id="43dc3-114">Namespace kanonischer Funktionen</span><span class="sxs-lookup"><span data-stu-id="43dc3-114">Canonical Functions Namespace</span></span>  
 <span data-ttu-id="43dc3-115">Der Namespace für kanonische Funktionen ist <xref:System.Data.Metadata.Edm>.</span><span class="sxs-lookup"><span data-stu-id="43dc3-115">The namespace for canonical function is <xref:System.Data.Metadata.Edm>.</span></span> <span data-ttu-id="43dc3-116">Der <xref:System.Data.Metadata.Edm>-Namespace wird in allen Abfragen automatisch eingebunden.</span><span class="sxs-lookup"><span data-stu-id="43dc3-116">The <xref:System.Data.Metadata.Edm> namespace is automatically included in all queries.</span></span> <span data-ttu-id="43dc3-117">Wenn jedoch ein anderer Namespace importiert wird, der eine Funktion mit dem gleichen Namen wie eine kanonische Funktion (im <xref:System.Data.Metadata.Edm>-Namespace) enthält, muss der Namespace angegeben werden.</span><span class="sxs-lookup"><span data-stu-id="43dc3-117">However, if another namespace is imported that contains a function with the same name as a canonical function (in the <xref:System.Data.Metadata.Edm> namespace), you must specify the namespace.</span></span>
