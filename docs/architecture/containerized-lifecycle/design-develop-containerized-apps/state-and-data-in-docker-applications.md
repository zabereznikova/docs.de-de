# <a name="state-and-data-in-docker-applications"></a><span data-ttu-id="2f6f4-103">Zustand und Daten in Docker-Anwendungen</span><span class="sxs-lookup"><span data-stu-id="2f6f4-103">State and data in Docker applications</span></span>

<span data-ttu-id="2f6f4-104">In den meisten Fällen können Sie sich einen Container als Instanz eines Prozesses vorstellen.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-104">In most cases, you can think of a container as an instance of a process.</span></span> <span data-ttu-id="2f6f4-105">Ein Prozess behält keinen persistenten Zustand bei.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-105">A process does not maintain persistent state.</span></span> <span data-ttu-id="2f6f4-106">Während ein Container in den lokalen Speicher schreiben kann, gleicht die Annahme, dass eine Instanz auf unbestimmte Zeit vorhanden ist, der Annahme, dass ein Speicherort im Arbeitsspeicher dauerhaft ist.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-106">While a container can write to its local storage, assuming that an instance will be around indefinitely is like assuming that a single location in memory will be durable.</span></span> <span data-ttu-id="2f6f4-107">Für Containerimages, z.B. Prozesse, sollte angenommen werden, dass sie über mehrere Instanzen verfügen und irgendwann abgebrochen werden. <span data-ttu-id="2f6f4-108">Die folgenden Lösungen werden verwendet, um persistente Daten in Docker-Anwendungen zu verwalten:</span><span class="sxs-lookup"><span data-stu-id="2f6f4-108">The following solutions are used to manage persistent data in Docker applications:</span></span>

<span data-ttu-id="2f6f4-109">Über den Docker-Host als [Docker-Volumes](https://docs.docker.com/engine/admin/volumes/):</span><span class="sxs-lookup"><span data-stu-id="2f6f4-109">From the Docker host, as [Docker Volumes](https://docs.docker.com/engine/admin/volumes/):</span></span>

- <span data-ttu-id="2f6f4-110">**Volumes** werden in einem Bereich des Hostdateisystems gespeichert, der von Docker verwaltet wird.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-110">**Volumes** are stored in an area of the host filesystem that's managed by Docker.</span></span>

- <span data-ttu-id="2f6f4-111">**Einbindungen über „Bind“** können jedem Ordner im Hostdateisystem zugeordnet werden, sodass der Zugriff nicht von einem Docker-Prozess gesteuert werden kann – dies wäre ein Sicherheitsrisiko, weil ein Container auf vertrauliche Betriebssystemordner zugreifen könnte.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-111">**Bind mounts** can map to any folder in the host filesystem, so access can't be controlled from a Docker process and can pose a security risk as a container could access sensitive OS folders.</span></span>

- <span data-ttu-id="2f6f4-112">**Einbindungen über tmpfs** sind ähnlich wie virtuelle Ordner, die nur im Arbeitsspeicher des Hosts vorhanden sind und nie in das Dateisystem geschrieben werden.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-112">**tmpfs mounts** are like virtual folders that only exist in the host's memory and are never written to the filesystem.</span></span>

<span data-ttu-id="2f6f4-113">Über den Remotespeicher:</span><span class="sxs-lookup"><span data-stu-id="2f6f4-113">From remote storage:</span></span>

- <span data-ttu-id="2f6f4-114">[Azure Storage](https://azure.microsoft.com/documentation/services/storage/), stellt geografisch verteilbaren Speicher und eine geeignete Lösung für langfristige Persistenz für Containern bereit.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-114">[Azure Storage](https://azure.microsoft.com/documentation/services/storage/) provides geo-distributable storage, providing a good long-term persistence solution for containers.</span></span>

- <span data-ttu-id="2f6f4-115">Relationale Remotedatenbanken wie die [Azure SQL-Datenbank](https://azure.microsoft.com/services/sql-database/), NoSQL-Datenbanken wie [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction) oder Zwischenspeicherdienste wie [Redis](https://redis.io/).</span><span class="sxs-lookup"><span data-stu-id="2f6f4-115">Remote relational databases like [Azure SQL Database](https://azure.microsoft.com/services/sql-database/), NoSQL databases like [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction), or cache services like [Redis](https://redis.io/).</span></span>

<span data-ttu-id="2f6f4-116">Über den Docker-Container:</span><span class="sxs-lookup"><span data-stu-id="2f6f4-116">From the Docker container:</span></span>

- <span data-ttu-id="2f6f4-117">Docker bietet ein Feature mit dem Namen *Overlay File System* (Dateisystem überlagern).</span><span class="sxs-lookup"><span data-stu-id="2f6f4-117">Docker provides a feature named the *overlay file system*.</span></span> <span data-ttu-id="2f6f4-118">Durch dieses Feature wird ein copy-on-write-Task (Bei Schreibvorgang kopieren) implementiert, der aktualisierte Informationen in das Stammdateisystem des Containers speichert.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-118">This feature implements a copy-on-write task that stores updated information to the root file system of the container.</span></span> <span data-ttu-id="2f6f4-119">Diese Informationen „liegen oberhalb“ des ursprünglichen Images, auf dem der Container basiert.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-119">That information "lays on top of" the original image on which the container is based.</span></span> <span data-ttu-id="2f6f4-120">Wenn der Container aus dem System gelöscht wird, gehen diese Änderungen verloren.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-120">If the container is deleted from the system, those changes are lost.</span></span> <span data-ttu-id="2f6f4-121">Aus diesem Grund ist es zwar möglich, den Zustand eines Containers in seinem lokalen Speicher zu speichern, doch das Entwerfen eines Systems auf der Grundlage dieses Features würde mit der Prämisse des Containerdesigns kollidieren, das standardmäßig zustandslos ist.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-121">Therefore, while it's possible to save the state of a container within its local storage, designing a system based on this feature would conflict with the premise of container design, which by default is stateless.</span></span>

- <span data-ttu-id="2f6f4-122">Docker-Volumes stellen aber mittlerweile das bevorzugte Verfahren zum Behandeln von lokalen Daten in Docker dar.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-122">However, Docker Volumes is now the preferred way to handle local data in Docker.</span></span> <span data-ttu-id="2f6f4-123">Wenn Sie weitere Informationen zum Speichern in Containern benötigen, lesen Sie [Docker storage drivers](https://docs.docker.com/engine/userguide/storagedriver/) (Docker-Speichertreiber) und [About images, containers, and storage drivers](https://docs.docker.com/engine/userguide/storagedriver/imagesandcontainers/) (Über Images, Container und Speichertreiber).</span><span class="sxs-lookup"><span data-stu-id="2f6f4-123">If you need more information about storage in containers, check on [Docker storage drivers](https://docs.docker.com/engine/userguide/storagedriver/) and [About images, containers, and storage drivers](https://docs.docker.com/engine/userguide/storagedriver/imagesandcontainers/).</span></span>

<span data-ttu-id="2f6f4-124">Nachfolgend finden Sie weitere Details zu diesen Optionen.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-124">The following provides additional detail about these options.</span></span>

<span data-ttu-id="2f6f4-125">**Volumes** sind Verzeichnisse, die vom Hostbetriebssystem den Verzeichnissen in Containern zugeordnet werden.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-125">**Volumes** are directories mapped from the host OS to directories in containers.</span></span> <span data-ttu-id="2f6f4-126">Wenn Code im Container Zugriff auf das Verzeichnis hat, ist dies ein Zugriff auf ein Verzeichnis auf dem Hostbetriebssystem.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-126">When code in the container has access to the directory, that access is actually to a directory on the host OS.</span></span> <span data-ttu-id="2f6f4-127">Dieses Verzeichnis ist nicht an die Lebensdauer des Containers selbst gebunden, und es wird von Docker verwaltet und von der Kernfunktionalität des Hostcomputers isoliert.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-127">This directory is not tied to the lifetime of the container itself, and the directory is managed by Docker and isolated from the core functionality of the host machine.</span></span> <span data-ttu-id="2f6f4-128">Deshalb sind diese Datenvolumes dazu entworfen, Daten unabhängig von der Lebensdauer des Containers beizubehalten.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-128">Thus, data volumes are designed to persist data independently of the life of the container.</span></span> <span data-ttu-id="2f6f4-129">Wenn Sie einen Container oder ein Image aus dem Docker-Host löschen, werden die persistenten Daten im Datenvolume nicht gelöscht.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-129">If you delete a container or an image from the Docker host, the data persisted in the data volume is not deleted.</span></span>

<span data-ttu-id="2f6f4-130">Volumes können benannt oder anonym sein (letzteres ist die Standardeinstellung).</span><span class="sxs-lookup"><span data-stu-id="2f6f4-130">Volumes can be named or anonymous (the default).</span></span> <span data-ttu-id="2f6f4-131">Benannte Volumes sind eine Weiterentwicklung von **Datenvolumecontainern** und vereinfachen die gemeinsame Nutzung von Daten durch Container.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-131">Named volumes are the evolution of **Data Volume Containers** and make it easy to share data between containers.</span></span> <span data-ttu-id="2f6f4-132">Volumes unterstützen auch Volumetreiber, mit denen Sie u.a. Daten auf Remotehosts speichern können.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-132">Volumes also support volume drivers that allow you to store data on remote hosts, among other options.</span></span>

<span data-ttu-id="2f6f4-133">**Einbindungen über „Bind“** sind schon lange verfügbar und ermöglichen die Zuordnung jedes Ordners zu einem Bereitstellungspunkt in einem Container.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-133">**Bind mounts** have been available for a long time and allow the mapping of any folder to a mount point in a container.</span></span> <span data-ttu-id="2f6f4-134">Einbindungen über „Bind“ weisen mehr Einschränkungen als Volumes sowie einige erhebliche Sicherheitsprobleme auf, daher sind Volumes die empfohlene Option.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-134">Bind mounts have more limitations than volumes and some important security issues, so volumes are the recommended option.</span></span>

<span data-ttu-id="2f6f4-135">**Einbindungen über `tmpfs`** sind virtuelle Ordner, die nur im Arbeitsspeicher des Hosts vorhanden sind und nie in das Dateisystem geschrieben werden.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-135">**`tmpfs` mounts** are virtual folders that live only in the host's memory and are never written to the filesystem.</span></span> <span data-ttu-id="2f6f4-136">Sie sind schnell und sicher, verbrauchen aber Arbeitsspeicher und sind nur für nicht persistente Daten vorgesehen.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-136">They are fast and secure but use memory and are only meant for non-persistent data.</span></span>

<span data-ttu-id="2f6f4-137">Wie in Abbildung 4-5 gezeigt, können reguläre Docker-Volumes außerhalb der Container selbst aber innerhalb der physischen Grenzen des Hostservers oder der VM gespeichert werden.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-137">As shown in Figure 4-5, regular Docker volumes can be stored outside of the containers themselves but within the physical boundaries of the host server or VM.</span></span> <span data-ttu-id="2f6f4-138">Docker-Container können jedoch nicht von einem Hostserver oder einer VM auf ein Volume zugreifen.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-138">However, Docker containers cannot access a volume from one host server or VM to another.</span></span> <span data-ttu-id="2f6f4-139">Anders gesagt: Bei diesen Volumes ist es nicht möglich, Daten zu verwalten, die von auf verschiedenen Docker-Hosts ausgeführten Containern gemeinsam verwendet werden. Dieses Szenario ließe sich allerdings mit einem Volumetreiber umsetzen, der Remotehosts unterstützt.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-139">In other words, with these volumes, it isn't possible to manage data shared between containers that run on different Docker hosts, although it could be achieved with a volume driver that supports remote hosts.</span></span>

<span data-ttu-id="2f6f4-146">Tools für **Remotedatenquellen und -caches** wie Azure SQL-Datenbank, Azure Cosmos DB oder ein Remotecache wie Redis können in Containeranwendungen auf die gleiche Weise verwendet werden wie bei der Entwicklung ohne Container.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-146">**Remote data sources and cache** tools like Azure SQL Database, Azure Cosmos DB, or a remote cache like Redis can be used in containerized applications the same way they are used when developing without containers.</span></span> <span data-ttu-id="2f6f4-147">Dies ist eine bewährte Möglichkeit zum Speichern von Geschäftsanwendungsdaten.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-147">This is a proven way to store business application data.</span></span>

<span data-ttu-id="2f6f4-148">**Azure Storage.**</span><span class="sxs-lookup"><span data-stu-id="2f6f4-148">**Azure Storage.**</span></span> <span data-ttu-id="2f6f4-149">Geschäftsdaten müssen in der Regel in externen Ressourcen oder Datenbanken wie Azure Storage abgelegt werden.</span><span class="sxs-lookup"><span data-stu-id="2f6f4-149">Business data usually needs to be placed in external resources or databases, like Azure Storage.</span></span> <span data-ttu-id="2f6f4-150">Azure Storage bietet die folgenden Dienste in der Cloud:</span><span class="sxs-lookup"><span data-stu-id="2f6f4-150">Azure Storage provides the following services in the cloud:</span></span>
