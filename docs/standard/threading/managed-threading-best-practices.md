### <a name="deadlocks"></a><span data-ttu-id="0bf68-112">Deadlocks</span><span class="sxs-lookup"><span data-stu-id="0bf68-112">Deadlocks</span></span>  
 <span data-ttu-id="0bf68-113">Ein Deadlock liegt vor, wenn zwei Threads gleichzeitig versuchen, eine Ressource zu sperren, die der jeweils andere Thread bereits gesperrt hat.</span><span class="sxs-lookup"><span data-stu-id="0bf68-113">A deadlock occurs when each of two threads tries to lock a resource the other has already locked.</span></span> <span data-ttu-id="0bf68-114">Keiner der beiden Threads kann weiter fortgesetzt werden.</span><span class="sxs-lookup"><span data-stu-id="0bf68-114">Neither thread can make any further progress.</span></span>  
  
 <span data-ttu-id="0bf68-115">Viele Methoden der Klassen des verwalteten Threadings stellen Timeouts bereit, mit deren Hilfe Sie Deadlocks entdecken können.</span><span class="sxs-lookup"><span data-stu-id="0bf68-115">Many methods of the managed threading classes provide time-outs to help you detect deadlocks.</span></span> <span data-ttu-id="0bf68-116">Im folgenden Code wird beispielsweise versucht, ein Objekt namens `lockObject` zu sperren.</span><span class="sxs-lookup"><span data-stu-id="0bf68-116">For example, the following code attempts to acquire a lock on an object named `lockObject`.</span></span> <span data-ttu-id="0bf68-117">Wenn die Sperrung nicht innerhalb von 300 Millisekunden erfolgt, gibt <xref:System.Threading.Monitor.TryEnter%2A?displayProperty=nameWithType> den Wert `false` zurück.</span><span class="sxs-lookup"><span data-stu-id="0bf68-117">If the lock is not obtained in 300 milliseconds, <xref:System.Threading.Monitor.TryEnter%2A?displayProperty=nameWithType> returns `false`.</span></span>  
  
```vb  
If Monitor.TryEnter(lockObject, 300) Then  
    Try  
        ' Place code protected by the Monitor here.  
    Finally  
        Monitor.Exit(lockObject)  
    End Try  
Else  
    ' Code to execute if the attempt times out.  
End If  
```  
  
```csharp  
if (Monitor.TryEnter(lockObject, 300)) {  
    try {  
        // Place code protected by the Monitor here.  
    }  
    finally {  
        Monitor.Exit(lockObject);  
    }  
}  
else {  
    // Code to execute if the attempt times out.  
}  
```  
  
### <a name="race-conditions"></a><span data-ttu-id="0bf68-118">Racebedingungen</span><span class="sxs-lookup"><span data-stu-id="0bf68-118">Race conditions</span></span>  
 <span data-ttu-id="0bf68-119">Bei einer Racebedingung handelt es sich um einen Fehler, der auftritt, wenn das Ergebnis eines Programms davon abhängt, welcher von zwei oder mehr Threads einen bestimmten Codeblock zuerst erreicht.</span><span class="sxs-lookup"><span data-stu-id="0bf68-119">A race condition is a bug that occurs when the outcome of a program depends on which of two or more threads reaches a particular block of code first.</span></span> <span data-ttu-id="0bf68-120">Wenn Sie das Programm mehrmals ausführen, werden verschiedene Ergebnisse erzeugt, und das Ergebnis einer bestimmten Ausführung lässt sich nicht vorhersagen.</span><span class="sxs-lookup"><span data-stu-id="0bf68-120">Running the program many times produces different results, and the result of any given run cannot be predicted.</span></span>  
  
 <span data-ttu-id="0bf68-121">Ein einfaches Beispiel einer Racebedingung ist die Erhöhung eines Feldes.</span><span class="sxs-lookup"><span data-stu-id="0bf68-121">A simple example of a race condition is incrementing a field.</span></span> <span data-ttu-id="0bf68-122">Angenommen, eine Klasse besitzt ein privates **static** -Feld ( **Shared** in Visual Basic), das jedes Mal, wenn eine Instanz der Klasse erstellt wird, mit Code wie `objCt++;` (C#) oder `objCt += 1` (Visual Basic) erhöht wird.</span><span class="sxs-lookup"><span data-stu-id="0bf68-122">Suppose a class has a private **static** field ( **Shared** in Visual Basic) that is incremented every time an instance of the class is created, using code such as `objCt++;` (C#) or `objCt += 1` (Visual Basic).</span></span> <span data-ttu-id="0bf68-123">Für diesen Vorgang muss der Wert von `objCt` in ein Register geladen, der Wert erhöht und in `objCt` gespeichert werden.</span><span class="sxs-lookup"><span data-stu-id="0bf68-123">This operation requires loading the value from `objCt` into a register, incrementing the value, and storing it in `objCt`.</span></span>  
  
 <span data-ttu-id="0bf68-124">In einer Multithreadanwendung kann ein Thread, der den Wert geladen und erhöht hat, von einem anderen Thread präemptiv unterbrochen werden, der dann alle drei Schritte ausführt.  <span data-ttu-id="0bf68-125">Diese spezielle Racebedingung lässt sich mit den Methoden der <xref:System.Threading.Interlocked>-Klasse (z. B. <xref:System.Threading.Interlocked.Increment%2A?displayProperty=nameWithType>) problemlos vermeiden.</span><span class="sxs-lookup"><span data-stu-id="0bf68-125">This particular race condition is easily avoided by using methods of the <xref:System.Threading.Interlocked> class, such as <xref:System.Threading.Interlocked.Increment%2A?displayProperty=nameWithType>.</span></span> <span data-ttu-id="0bf68-126">Weitere Techniken zum Synchronisieren von Daten zwischen mehreren Threads finden Sie unter [Synchronizing Data for Multithreading (Synchronisieren von Daten für Multithreading)](synchronizing-data-for-multithreading.md).</span><span class="sxs-lookup"><span data-stu-id="0bf68-126">To read about other techniques for synchronizing data among multiple threads, see [Synchronizing Data for Multithreading](synchronizing-data-for-multithreading.md).</span></span>  
  
 <span data-ttu-id="0bf68-127">Racebedingungen können auch auftreten, wenn Sie die Aktivitäten von mehreren Threads synchronisieren.</span><span class="sxs-lookup"><span data-stu-id="0bf68-127">Race conditions can also occur when you synchronize the activities of multiple threads.</span></span> <span data-ttu-id="0bf68-128">Bei jeder Codezeile, die Sie schreiben, müssen Sie sich überlegen, was passieren kann, wenn ein Thread vor der Ausführung der Zeile (oder jeder einzelnen Anweisung, aus denen die Zeile besteht) präemptiv unterbrochen und die Ausführung von einem anderen Thread fortgesetzt wird.</span><span class="sxs-lookup"><span data-stu-id="0bf68-128">Whenever you write a line of code, you must consider what might happen if a thread were preempted before executing the line (or before any of the individual machine instructions that make up the line), and another thread overtook it.</span></span>
