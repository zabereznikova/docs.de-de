# <a name="microservices-addressability-and-the-service-registry"></a><span data-ttu-id="50e35-103">Adressierbarkeit von Microservices und die Dienstregistrierung</span><span class="sxs-lookup"><span data-stu-id="50e35-103">Microservices addressability and the service registry</span></span>

<span data-ttu-id="50e35-104">Jeder Microservice hat einen eindeutigen Namen (URL), der verwendet wird, um den Speicherort aufzulösen.</span><span class="sxs-lookup"><span data-stu-id="50e35-104">Each microservice has a unique name (URL) that is used to resolve its location.</span></span> <span data-ttu-id="50e35-105">Ihr Microservice muss adressierbar sein, unabhängig davon, wo er ausgeführt wird.</span><span class="sxs-lookup"><span data-stu-id="50e35-105">Your microservice needs to be addressable wherever it is running.</span></span> <span data-ttu-id="50e35-106">Wenn Sie darüber nachdenken müssen, welcher Computer einen bestimmten Microservice ausführt, können schnell Fehler auftreten.</span><span class="sxs-lookup"><span data-stu-id="50e35-106">If you have to think about which computer is running a particular microservice, things can go bad quickly.</span></span> <span data-ttu-id="50e35-107">Ähnlich wie DNS eine URL zu einem bestimmten Computer auflöst, benötigt Ihr Microservice einen eindeutigen Namen, damit seine aktuelle Position ermittelt werden kann.</span><span class="sxs-lookup"><span data-stu-id="50e35-107">In the same way that DNS resolves a URL to a particular computer, your microservice needs to have a unique name so that its current location is discoverable.</span></span> <span data-ttu-id="50e35-108">Microservices benötigen adressierbare Namen, um von der Infrastruktur unabhängig zu sein, auf der sie ausgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="50e35-108">Microservices need addressable names that make them independent from the infrastructure that they are running on.</span></span> <span data-ttu-id="50e35-109">Da eine [Dienstregistrierung](https://microservices.io/patterns/service-registry.html) vorhanden sein muss, impliziert dies, dass eine Interaktion zwischen der Bereitstellung und der Erkennung besteht.</span><span class="sxs-lookup"><span data-stu-id="50e35-109">This implies that there is an interaction between how your service is deployed and how it is discovered, because there needs to be a [service registry](https://microservices.io/patterns/service-registry.html).</span></span> <span data-ttu-id="50e35-110">In gleicher Weise muss der Registrierungsdienst angeben können, wo der Dienst momentan ausgeführt wird, wenn bei einem Computer ein Fehler auftritt.</span><span class="sxs-lookup"><span data-stu-id="50e35-110">In the same vein, when a computer fails, the registry service must be able to indicate where the service is now running.</span></span>

<span data-ttu-id="50e35-111">Das [Muster der Dienstregistrierung](https://microservices.io/patterns/service-registry.html) ist ein Schlüsselteil der Diensterkennung.</span><span class="sxs-lookup"><span data-stu-id="50e35-111">The [service registry pattern](https://microservices.io/patterns/service-registry.html) is a key part of service discovery.</span></span> <span data-ttu-id="50e35-112">Die Registrierung ist eine Datenbank, die Netzwerkadressen und Dienstinstanzen enthält.</span><span class="sxs-lookup"><span data-stu-id="50e35-112">The registry is a database containing the network locations of service instances.</span></span> <span data-ttu-id="50e35-113">Eine Dienstregistrierung muss hoch verfügbar und auf dem neuesten Stand sein.</span><span class="sxs-lookup"><span data-stu-id="50e35-113">A service registry needs to be highly available and up to date.</span></span> <span data-ttu-id="50e35-114">Clients könnten Netzwerkadressen zwischenspeichern, die sie von der Dienstregistrierung erhalten.</span><span class="sxs-lookup"><span data-stu-id="50e35-114">Clients could cache network locations obtained from the service registry.</span></span> <span data-ttu-id="50e35-115">Allerdings veralten diese Informationen, und Clients können die Dienstinstanzen nicht mehr ermitteln.</span><span class="sxs-lookup"><span data-stu-id="50e35-115">However, that information eventually goes out of date and clients can no longer discover service instances.</span></span> <span data-ttu-id="50e35-116">Folglich besteht eine Dienstregistrierung aus einem Cluster von Servern, die ein Replikationsprotokoll verwenden, um die Konsistenz zu gewährleisten.</span><span class="sxs-lookup"><span data-stu-id="50e35-116">Consequently, a service registry consists of a cluster of servers that use a replication protocol to maintain consistency.</span></span>

<span data-ttu-id="50e35-117">In manchen Bereitstellungsumgebungen für Microservices (auch Clusters genannt, wie in einem späteren Abschnitt erklärt wird) ist die Diensterkennung integriert.</span><span class="sxs-lookup"><span data-stu-id="50e35-117">In some microservice deployment environments (called clusters, to be covered in a later section), service discovery is built-in.</span></span> <span data-ttu-id="50e35-118">Zum Beispiel können Kubernetes und DC/OS mit Marathon innerhalb einer Azure Container Service-Umgebung die Registrierung und die Registrierungsaufhebung von Dienstinstanzen verarbeiten.</span><span class="sxs-lookup"><span data-stu-id="50e35-118">For example, within an Azure Container Service environment, Kubernetes and DC/OS with Marathon can handle service instance registration and deregistration.</span></span> <span data-ttu-id="50e35-119">Diese führen ebenfalls einen Proxy auf jedem Clusterhost aus, der die Rolle eines serverseitigen Suchrouters übernimmt.</span><span class="sxs-lookup"><span data-stu-id="50e35-119">They also run a proxy on each cluster host that plays the role of server-side discovery router.</span></span> <span data-ttu-id="50e35-120">Ein weiteres Beispiel ist Azure Service Fabric, was ebenfalls eine Dienstregistrierung über einen integrierten Namensdienst bereitstellt.</span><span class="sxs-lookup"><span data-stu-id="50e35-120">Another example is Azure Service Fabric, which also provides a service registry through its out-of-the-box Naming Service.</span></span>

<span data-ttu-id="50e35-121">Beachten Sie, dass es eine gewisse Überschneidung zwischen der Dienstregistrierung und dem API-Gatewaymuster gibt, die ebenfalls zur Lösung dieses Problems beiträgt.</span><span class="sxs-lookup"><span data-stu-id="50e35-121">Note that there is certain overlap between the service registry and the API gateway pattern, which helps solve this problem as well.</span></span> <span data-ttu-id="50e35-122">Zum Beispiel ist der [Reverseproxy in Service Fabric](https://docs.microsoft.com/azure/service-fabric/service-fabric-reverseproxy) eine Art der Implementierung eines API-Gateways, die auf dem Namensdienst von Service Fabric basiert und dabei hilft, die Adressauflösung in interne Dienste aufzulösen.</span><span class="sxs-lookup"><span data-stu-id="50e35-122">For example, the [Service Fabric Reverse Proxy](https://docs.microsoft.com/azure/service-fabric/service-fabric-reverseproxy) is a type of implementation of an API Gateway that is based on the Service Fabrice Naming Service and that helps resolve address resolution to the internal services.</span></span>
