# <a name="runtime-callable-wrapper"></a><span data-ttu-id="32b6b-103">Runtime Callable Wrapper (RCW)</span><span class="sxs-lookup"><span data-stu-id="32b6b-103">Runtime Callable Wrapper</span></span>

<span data-ttu-id="32b6b-104">Die Common Language Runtime macht COM-Objekte über einen Proxy verfügbar, der RCW (Runtime Callable Wrapper, Aufrufwrapper der Common Language Runtime) genannt wird.</span><span class="sxs-lookup"><span data-stu-id="32b6b-104">The common language runtime exposes COM objects through a proxy called the runtime callable wrapper (RCW).</span></span> <span data-ttu-id="32b6b-105">Obwohl .NET-Clients einen RCW als normales Objekt betrachten, besteht seine primäre Funktion im Marshallen von Aufrufen zwischen einem .NET-Client und einem COM-Objekt.</span><span class="sxs-lookup"><span data-stu-id="32b6b-105">Although the RCW appears to be an ordinary object to .NET clients, its primary function is to marshal calls between a .NET client and a COM object.</span></span>  
  
 <span data-ttu-id="32b6b-106">Die Common Language Runtime erstellt genau einen RCW für jedes COM-Objekt, unabhängig von der Anzahl der vorhandenen Verweise auf das Objekt.</span><span class="sxs-lookup"><span data-stu-id="32b6b-106">The runtime creates exactly one RCW for each COM object, regardless of the number of references that exist on that object.</span></span> <span data-ttu-id="32b6b-107">Die Common Language Runtime verwaltet für jedes Objekt einen einzelnen RCW pro Prozess.</span><span class="sxs-lookup"><span data-stu-id="32b6b-107">The runtime maintains a single RCW per process for each object.</span></span>  <span data-ttu-id="32b6b-108">Wenn Sie einen RCW in einer Anwendungsdomäne oder einem Apartment erstellen und anschließend einen Verweis an eine andere Anwendungsdomäne oder ein anderes Apartment übergeben, wird ein Proxy für das erste Objekt verwendet.</span><span class="sxs-lookup"><span data-stu-id="32b6b-108">If you create an RCW in one application domain or apartment, and then pass a reference to another application domain or apartment, a proxy to the first object will be used.</span></span>  <span data-ttu-id="32b6b-109">Wie die folgende Abbildung zeigt, kann eine beliebige Anzahl verwalteter Clients auf die COM-Objekte verweisen, welche die Schnittstellen "INew" und "INewer" verfügbar machen.</span><span class="sxs-lookup"><span data-stu-id="32b6b-109">As the following illustration shows, any number of managed clients can hold a reference to the COM objects that expose INew and INewer interfaces.</span></span>

 <span data-ttu-id="32b6b-112">Mithilfe von Metadaten, die aus einer Typbibliothek abgeleitet werden, erstellt die Common Language Runtime sowohl das aufzurufende COM-Objekt als auch einen Wrapper für dieses Objekt.</span><span class="sxs-lookup"><span data-stu-id="32b6b-112">Using metadata derived from a type library, the runtime creates both the COM object being called and a wrapper for that object.</span></span> <span data-ttu-id="32b6b-113">Jeder RCW verwaltet einen Cache von Schnittstellenzeigern auf das umschlossene COM-Objekt.  <span data-ttu-id="32b6b-115">Unter anderem marshallt der RCW für das umschlossene Objekt Daten zwischen verwaltetem und nicht verwaltetem Code.</span><span class="sxs-lookup"><span data-stu-id="32b6b-115">Among other activities, the RCW marshals data between managed and unmanaged code, on behalf of the wrapped object.</span></span> <span data-ttu-id="32b6b-116">Insbesondere marshallt der RCW Methodenargumente und von Methoden zurückgegebene Werte, wenn der Client und der Server die untereinander ausgetauschten Daten unterschiedlich darstellen.</span><span class="sxs-lookup"><span data-stu-id="32b6b-116">Specifically, the RCW provides marshaling for method arguments and method return values whenever the client and server have different representations of the data passed between them.</span></span>
