<span data-ttu-id="e929f-103">Da eine Webseite in verschiedensten Ländern auf der ganzen Welt angezeigt werden kann, können Benutzer numerische Daten in einer nahezu unbegrenzten Anzahl von Formaten in ein <xref:System.Web.UI.WebControls.TextBox>-Steuerelement einfügen.</span><span class="sxs-lookup"><span data-stu-id="e929f-103">Because a Web page can be displayed anywhere in the world, users can input numeric data into a <xref:System.Web.UI.WebControls.TextBox> control in an almost unlimited number of formats.</span></span> <span data-ttu-id="e929f-104">Daher ist es von entscheidender Bedeutung, das Gebietsschema und die Kultur des Webseitenbenutzers zu ermitteln.</span><span class="sxs-lookup"><span data-stu-id="e929f-104">As a result, it is very important to determine the locale and culture of the Web page's user.</span></span> <span data-ttu-id="e929f-105">Bei der Analyse von Benutzereingaben können Sie dann die vom Gebietsschema und der Kultur des Benutzers definierten Formatierungskonventionen anwenden.</span><span class="sxs-lookup"><span data-stu-id="e929f-105">When you parse user input, you can then apply the formatting conventions defined by the user's locale and culture.</span></span>  
  
### <a name="to-convert-numeric-input-from-a-web-textbox-control-to-a-number"></a><span data-ttu-id="e929f-106">So konvertieren Sie numerische Eingaben eines TextBox-Websteuerelements in eine Zahl</span><span class="sxs-lookup"><span data-stu-id="e929f-106">To convert numeric input from a Web TextBox control to a number</span></span>  
  
1.  <span data-ttu-id="e929f-107">Stellen Sie fest, ob das von der <xref:System.Web.HttpRequest.UserLanguages%2A?displayProperty=nameWithType>-Eigenschaft zurückgegebene Zeichenfolgenarray aufgefüllt ist.</span><span class="sxs-lookup"><span data-stu-id="e929f-107">Determine whether the string array returned by the <xref:System.Web.HttpRequest.UserLanguages%2A?displayProperty=nameWithType> property is populated.</span></span> <span data-ttu-id="e929f-108">Wenn dies nicht der Fall ist, fahren Sie mit Schritt 6 fort.</span><span class="sxs-lookup"><span data-stu-id="e929f-108">If it is not, continue to step 6.</span></span>  
  
2.  <span data-ttu-id="e929f-109">Wenn das von der <xref:System.Web.HttpRequest.UserLanguages%2A>-Eigenschaft zurückgegebene Zeichenfolgenarray aufgefüllt ist, rufen Sie sein erstes Element ab.</span><span class="sxs-lookup"><span data-stu-id="e929f-109">If the string array returned by the <xref:System.Web.HttpRequest.UserLanguages%2A> property is populated, retrieve its first element.</span></span> <span data-ttu-id="e929f-110">Das erste Element gibt die Standardeinstellung des Benutzers oder seine bevorzugte Sprache und Region an.</span><span class="sxs-lookup"><span data-stu-id="e929f-110">The first element indicates the user's default or preferred language and region.</span></span>  
  
3.  <span data-ttu-id="e929f-111">Instanziieren Sie durch Aufrufen des <xref:System.Globalization.CultureInfo.%23ctor%28System.String%2CSystem.Boolean%29?displayProperty=nameWithType>-Konstruktors ein <xref:System.Globalization.CultureInfo>-Objekt, das die bevorzugte Kultur des Benutzers darstellt.</span><span class="sxs-lookup"><span data-stu-id="e929f-111">Instantiate a <xref:System.Globalization.CultureInfo> object that represents the user's preferred culture by calling the <xref:System.Globalization.CultureInfo.%23ctor%28System.String%2CSystem.Boolean%29?displayProperty=nameWithType> constructor.</span></span>  
  
4.  <span data-ttu-id="e929f-112">Rufen Sie entweder die `TryParse`- oder `Parse`-Methode des numerischen Typs auf, in den die Benutzereingabe konvertiert werden soll.</span><span class="sxs-lookup"><span data-stu-id="e929f-112">Call either the `TryParse` or the `Parse` method of the numeric type that you want to convert the user's input to.</span></span> <span data-ttu-id="e929f-113">Verwenden Sie eine Überladung der `TryParse`- oder `Parse`-Methode mit einem `provider`-Parameter, und übergeben Sie sie an eines der folgenden Objekte:</span><span class="sxs-lookup"><span data-stu-id="e929f-113">Use an overload of the `TryParse` or the `Parse` method with a `provider` parameter, and pass it either of the following:</span></span>  
  
    -   <span data-ttu-id="e929f-114">Das in Schritt 3 erstellte <xref:System.Globalization.CultureInfo>-Objekt</span><span class="sxs-lookup"><span data-stu-id="e929f-114">The <xref:System.Globalization.CultureInfo> object created in step 3.</span></span>  
  
    -   <span data-ttu-id="e929f-115">Das von der <xref:System.Globalization.CultureInfo.NumberFormat%2A>-Eigenschaft zurückgegebene <xref:System.Globalization.NumberFormatInfo>-Objekt des in Schritt 3 erstellten <xref:System.Globalization.CultureInfo>-Objekts</span><span class="sxs-lookup"><span data-stu-id="e929f-115">The <xref:System.Globalization.NumberFormatInfo> object that is returned by the <xref:System.Globalization.CultureInfo.NumberFormat%2A> property of the <xref:System.Globalization.CultureInfo> object created in step 3.</span></span>  
  
5.  <span data-ttu-id="e929f-116">Wenn bei der Konvertierung ein Fehler auftritt, wiederholen Sie die Schritte 2 bis 4 für jedes verbleibende Element im von der <xref:System.Web.HttpRequest.UserLanguages%2A>-Eigenschaft zurückgegebenen Zeichenfolgenarray.</span><span class="sxs-lookup"><span data-stu-id="e929f-116">If the conversion fails, repeat steps 2 through 4 for each remaining element in the string array returned by the <xref:System.Web.HttpRequest.UserLanguages%2A> property.</span></span>  
  
6.  <span data-ttu-id="e929f-117">Wenn bei der Konvertierung weiterhin ein Fehler auftritt, oder das von der <xref:System.Web.HttpRequest.UserLanguages%2A>-Eigenschaft zurückgegebene Zeichenfolgenarray leer ist, analysieren Sie die Zeichenfolge mit der invarianten Kultur, die durch die <xref:System.Globalization.CultureInfo.InvariantCulture%2A?displayProperty=nameWithType>-Eigenschaft zurückgegeben wird.</span><span class="sxs-lookup"><span data-stu-id="e929f-117">If the conversion still fails or if the string array returned by the <xref:System.Web.HttpRequest.UserLanguages%2A> property is empty, parse the string by using the invariant culture, which is returned by the <xref:System.Globalization.CultureInfo.InvariantCulture%2A?displayProperty=nameWithType> property.</span></span>
