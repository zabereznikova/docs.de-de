## <a name="easily-tested-with-automated-tests"></a><span data-ttu-id="aaa33-136">Problemloses Testen mit automatisierten Tests</span><span class="sxs-lookup"><span data-stu-id="aaa33-136">Easily tested with automated tests</span></span>

<span data-ttu-id="aaa33-137">ASP.NET Core-Anwendungen unterstützen Komponententests, und deren lose Kopplung und Unterstützung für Abhängigkeitsinjektionen erleichtert das Austauschen der Infrastrukturprobleme mit falschen Implementierungen zu Testzwecken.</span><span class="sxs-lookup"><span data-stu-id="aaa33-137">ASP.NET Core applications support unit testing, and their loose coupling and support for dependency injections makes it easy to swap infrastructure concerns with fake implementations for test purposes.</span></span> <span data-ttu-id="aaa33-138">ASP.NET Core umfasst auch einen TestServer, der verwendet werden kann, um Anwendungen im Arbeitsspeicher zu hosten.</span><span class="sxs-lookup"><span data-stu-id="aaa33-138">ASP.NET Core also ships a TestServer that can be used to host apps in memory.</span></span> <span data-ttu-id="aaa33-139">Funktionstests können dann Anforderungen an diesen In-Memory-Server stellen, den vollständigen Anwendungsstapel ausführen (einschließlich Middleware, Routing, Modellbindung, Filter usw.) und eine Antwort erhalten. ## <a name="traditional-and-spa-behaviors-supported"></a><span data-ttu-id="aaa33-141">Herkömmliches Verhalten und SPA-Verhalten unterstützt</span><span class="sxs-lookup"><span data-stu-id="aaa33-141">Traditional and SPA behaviors supported</span></span>

<span data-ttu-id="aaa33-142">Herkömmliche Webanwendungen verfügen über wenig clientseitiges Verhalten. Stattdessen verlassen sie sich bei allen Navigationen, Abfragen und Updates, die die Anwendung benötigen könnte, auf den Server.</span><span class="sxs-lookup"><span data-stu-id="aaa33-142">Traditional web applications have involved little client-side behavior, but instead have relied on the server for all navigation, queries, and updates the app might need to make.</span></span> <span data-ttu-id="aaa33-143">Jeder neue vom Benutzer ausgeführte Vorgang würde in eine neue Webanforderung verschoben werden. Das Ergebnis wäre ein vollständiges Neuladen der Seite im Browser des Benutzers.</span><span class="sxs-lookup"><span data-stu-id="aaa33-143">Each new operation made by the user would be translated into a new web request, with the result being a full page reload in the end user's browser.</span></span> <span data-ttu-id="aaa33-144">Klassische MVC-Frameworks folgen normalerweise diesem Verfahren, bei dem jede neue Anforderung zu einer anderen Controlleraktion gehört, die wiederum mit einem Modell arbeiten und eine Ansicht zurückgeben würde.</span><span class="sxs-lookup"><span data-stu-id="aaa33-144">Classic Model-View-Controller (MVC) frameworks typically follow this approach, with each new request corresponding to a different controller action, which in turn would work with a model and return a view.</span></span> <span data-ttu-id="aaa33-145">Einige individuelle Vorgänge auf einer vorhandenen Seite könnten mit der AJAX-Funktion (Asynchronous JavaScript And XML) erweitert werden, aber die Gesamtarchitektur der Anwendung hat viele verschiedene MVC-Ansichten und URL-Endpunkte verwendet.</span><span class="sxs-lookup"><span data-stu-id="aaa33-145">Some individual operations on a given page might be enhanced with AJAX (Asynchronous JavaScript and XML) functionality, but the overall architecture of the app used many different MVC views and URL endpoints.</span></span>

<span data-ttu-id="aaa33-146">Single-Page-Webanwendungen (SPAs) schließen dagegen wenige dynamisch erstellte serverseitige Ladevorgänge der Seite ein (sofern vorhanden).</span><span class="sxs-lookup"><span data-stu-id="aaa33-146">Single Page Applications (SPAs), by contrast, involve very few dynamically generated server-side page loads (if any).</span></span> <span data-ttu-id="aaa33-147">Viele SPAs werden innerhalb einer statischen HTML-Datei initialisiert, die die notwendigen JavaScript-Bibliotheken lädt, um die Anwendung zu starten und auszuführen.</span><span class="sxs-lookup"><span data-stu-id="aaa33-147">Many SPAs are initialized within a static HTML file which loads the necessary JavaScript libraries to start and run the app.</span></span> <span data-ttu-id="aaa33-148">Diese Anwendungen führen zu einer hohen Auslastung der Web-APIs für deren Datenanforderungen und können für mehr Benutzerfreundlichkeit sorgen.</span><span class="sxs-lookup"><span data-stu-id="aaa33-148">These apps make heavy usage of web APIs for their data needs, and can provide much richer user experiences.</span></span>

<span data-ttu-id="aaa33-149">Viele Webanwendungen verfügen über eine Kombination aus den Verhaltensweisen von herkömmlichen Webanwendungen (normalerweise für Inhalte) und SPAs (für Interaktivität).</span><span class="sxs-lookup"><span data-stu-id="aaa33-149">Many web applications involve a combination of traditional web application behavior (typically for content) and SPAs (for interactivity).</span></span> <span data-ttu-id="aaa33-150">ASP.NET Core unterstützt sowohl MVC-APIs (Ansichten und/oder Razor Pages) als auch Web-APIs in derselben Anwendung. Dabei werden das gleiche Toolset und die gleichen zugrunde liegenden Frameworkbibliotheken verwendet.</span><span class="sxs-lookup"><span data-stu-id="aaa33-150">ASP.NET Core supports both MVC (Views and/or Razor Pages) and web APIs in the same application, using the same set of tools and underlying framework libraries.</span></span>
