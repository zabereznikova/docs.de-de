<span data-ttu-id="25148-103">Die **ImportNode**-Methode ist das Verfahren, mit dem Sie einen Knoten oder eine ganze Knotenunterstruktur von einem **XmlDocument** in ein anderes kopieren können.</span><span class="sxs-lookup"><span data-stu-id="25148-103">The **ImportNode** method is the mechanism by which a node or entire node subtree is copied from one **XmlDocument** to another.</span></span> <span data-ttu-id="25148-104">Der Knoten, der von dem Aufruf zurückgegeben wird, ist eine Kopie des Knotens aus dem Quelldokument, einschließlich Attributwerten, Knotennamen, Knotentyp und allen namespacebezogenen Attributen wie Präfix, lokaler Name und Namespace-URI (Uniform Resource Identifier).</span><span class="sxs-lookup"><span data-stu-id="25148-104">The node returned from the call is a copy of the node from the source document, including attribute values, the node name, node type, and all namespace-related attributes such as the prefix, local name, and namespace Uniform Resource Identifier (URI).</span></span> <span data-ttu-id="25148-105">Das Quelldokument bleibt dabei unverändert.</span><span class="sxs-lookup"><span data-stu-id="25148-105">The source document is not changed.</span></span> <span data-ttu-id="25148-106">Nachdem Sie den Knoten importiert haben, müssen Sie ihn noch der Struktur hinzufügen, indem Sie eine der Methoden zum Einfügen von Knoten verwenden.</span><span class="sxs-lookup"><span data-stu-id="25148-106">After you have imported the node, you still have to add it to the tree using one of the methods used to insert nodes.</span></span>  
  
 <span data-ttu-id="25148-107">Sobald der Knoten an das neue Dokument angehängt worden ist, ist das neue Dokument Eigentümer des Knotens.</span><span class="sxs-lookup"><span data-stu-id="25148-107">When the node is attached to its new document, the new document owns the node.</span></span> <span data-ttu-id="25148-108">Grund hierfür ist, dass jeder Knoten, wenn er erstellt wird, zu einem Dokument gehört, das sein Eigentümer ist.
